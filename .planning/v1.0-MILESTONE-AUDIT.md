### Design Divergence: CHAT-05

**Spec:** "Trades specified by the LLM execute automatically — no confirmation dialog."
**Implementation:** Trades returned as `proposed_trades`; frontend renders `ProposedTradeCard` with Confirm/Dismiss; confirmation routes through same `execute_trade` service.
**Assessment:** Deliberate UX improvement. Full execution path wired end-to-end. Watchlist changes (CHAT-06) ARE auto-executed as specified.
**Severity:** Info — intentional, not a defect
