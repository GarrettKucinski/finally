4. **test/playwright.config.ts** -- Create the Playwright configuration:
```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: false,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: 1,
  reporter: [['html', { open: 'never' }], ['list']],
  timeout: 30000,

  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:3000',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },

  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
  ],
});
```

Key details:
- `fullyParallel: false` and `workers: 1` because tests share state (same database, same portfolio)
- `baseURL` reads from `BASE_URL` env var (set to `http://frontend:3000` in Docker, defaults to `http://localhost:3000` for local runs)
- Single project (chromium only) for fast execution
- `timeout: 30000` (30s) gives enough time for SSE to start streaming
- Retries in CI for flaky network/container startup timing

5. **test/docker-compose.test.yml** -- Create the test compose file that extends the main compose:
```yaml
include:
  - path: ../docker-compose.yml

services:
  backend:
    environment:
      - LLM_MOCK=true

  playwright:
    image: mcr.microsoft.com/playwright:v1.52.0-noble
    working_dir: /tests
    volumes:
      - ./:/tests
    environment:
      - BASE_URL=http://frontend:3000
      - CI=true
    depends_on:
      frontend:
        condition: service_started
    command: npx playwright test --project=chromium
```

IMPORTANT: The Playwright Docker image tag (v1.52.0) MUST match the @playwright/test version in package.json. If you used a different version in package.json, update this image tag to match.

Key details:
- `include` directive pulls in the main docker-compose.yml (both frontend and backend services)
- Backend service is overridden to add `LLM_MOCK=true` (env var injected alongside the existing env_file vars)
- Playwright container mounts the test directory and runs tests against `http://frontend:3000` (Docker internal network)
- `depends_on: frontend: condition: service_started` ensures the frontend container is running (frontend already depends on healthy backend via the main compose file, so transitively the whole stack is ready)
