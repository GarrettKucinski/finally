2. **frontend/Dockerfile** -- Create a 3-stage multi-stage Dockerfile:

**Stage 1 (deps):** `FROM node:22-slim AS deps`. WORKDIR /app. Copy package.json and package-lock.json. Run `npm ci`.

**Stage 2 (builder):** `FROM node:22-slim AS builder`. WORKDIR /app. Copy node_modules from deps stage. Copy all source files. Set `ENV NEXT_TELEMETRY_DISABLED=1`. Run `npm run build`.

**Stage 3 (runner):** `FROM node:22-slim AS runner`. WORKDIR /app. Set environment variables: `NODE_ENV=production`, `NEXT_TELEMETRY_DISABLED=1`, `HOSTNAME=0.0.0.0`, `PORT=3000`. Create a non-root user: `addgroup --system --gid 1001 nodejs` and `adduser --system --uid 1001 nextjs`. Copy `public/` from builder. Copy `.next/standalone` from builder (owned by nextjs:nodejs). Copy `.next/static` from builder into `.next/static` (owned by nextjs:nodejs). Switch to USER nextjs. EXPOSE 3000. CMD `["node", "server.js"]`.

CRITICAL: `HOSTNAME=0.0.0.0` must be set so the standalone server listens on all interfaces (Pitfall 2 from research). Static assets must be copied explicitly (Pitfall 1 from research).

4. **backend/Dockerfile** -- Create using the official uv base image:
```dockerfile
FROM ghcr.io/astral-sh/uv:python3.12-bookworm-slim

WORKDIR /app

ENV UV_COMPILE_BYTECODE=1
ENV UV_LINK_MODE=copy

# Install dependencies first (cached layer) -- no dev deps in production
RUN --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \
    --mount=type=bind,source=pyproject.toml,target=pyproject.toml \
    uv sync --locked --no-dev --no-install-project

# Copy application code (includes app/schema/*.sql files needed at import time)
COPY . /app
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --locked --no-dev

ENV PATH="/app/.venv/bin:$PATH"

EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Key details:
- `UV_COMPILE_BYTECODE=1` for faster startup
- `UV_LINK_MODE=copy` required when using cache mounts
- `--no-dev` excludes pytest, ruff, and other dev dependencies
- Two-step install: deps first (cached), then full project with source
- `app/schema/*.sql` files are included via `COPY . /app` (loaded at import time by pathlib)

1. **docker-compose.yml** -- Create in the project root with two services:

```yaml
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    env_file: .env
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/api/health')"]
      interval: 5s
      timeout: 5s
      retries: 5
      start_period: 10s

  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    environment:
      - BACKEND_URL=http://backend:8000
    depends_on:
      backend:
        condition: service_healthy
```

Key details:
- No `version:` key (deprecated in Compose v2)
- No explicit `networks:` block (Compose creates a default network automatically)
- Backend uses `env_file: .env` to inject environment variables (Pydantic Settings reads env vars first, before .env file -- Pitfall 3 from research)
- Backend healthcheck uses Python stdlib urllib (available in the container) to hit the existing `/api/health` endpoint
- Frontend sets `BACKEND_URL=http://backend:8000` which is used by both next.config.ts rewrites and the SSE route handler
- Frontend depends on backend being healthy before starting (Pitfall 7 from research)
- Backend is exposed on 8000 for debugging; frontend on 3000 as the user-facing port
