    Create `backend/app/main.py` — the FastAPI application entry point.

    **Lifespan context manager:**
    ```python
    from contextlib import asynccontextmanager
    from fastapi import FastAPI
    from .config import Settings
    from .db import init_db, close_db
    from .market.cache import PriceCache
    from .market.factory import create_market_data_source
    from .market.stream import create_stream_router

    @asynccontextmanager
    async def lifespan(app: FastAPI):
        # 1. Load settings
        settings = Settings()

        # 2. Initialize database (pool + schema + seed)
        pool = await init_db(settings.database_url)
        app.state.db_pool = pool
        app.state.settings = settings

        # 3. Load default watchlist tickers from DB
        async with pool.acquire() as conn:
            rows = await conn.fetch(
                "SELECT ticker FROM watchlist WHERE user_id = $1",
                "00000000-0000-0000-0000-000000000001"
            )
        tickers = [row["ticker"] for row in rows] if rows else []

        # 4. Start market data
        cache = PriceCache()
        source = create_market_data_source(cache)
        if tickers:
            await source.start(tickers)
        app.state.price_cache = cache
        app.state.market_source = source

        yield

        # Shutdown
        await source.stop()
        await close_db(pool)
    ```

    **App creation:**
    ```python
    app = FastAPI(title="FinAlly", lifespan=lifespan)
    ```

    **Router includes:**
    - The SSE stream router CANNOT use the factory pattern at module level because it needs the price_cache from app.state. Instead, include the stream router during the lifespan after creating the cache:
      ```python
      stream_router = create_stream_router(cache)
      app.include_router(stream_router)
      ```
      Place the `app.include_router(stream_router)` call inside the lifespan, after creating the cache but before `yield`.

    **Important details:**
    - Import `DEFAULT_WATCHLIST` from `market.seed_prices` only as a fallback if the DB query returns empty (shouldn't happen after seed, but defensive).
    - Store all shared state on `app.state`: `db_pool`, `settings`, `price_cache`, `market_source`.
    - Do NOT add the health route here -- that's Plan 01-02.
