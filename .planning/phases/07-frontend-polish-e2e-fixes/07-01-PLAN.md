<task type="auto">
  <name>Task 2: Add stopPropagation to watchlist remove button</name>
  <files>frontend/src/components/watchlist/WatchlistPanel.tsx</files>
  <action>
In `WatchlistPanel.tsx`, find the remove button's `onClick` handler inside the `WatchlistRow` component (line ~65). Change:

```tsx
onClick={() => onRemove(ticker)}
```

to:

```tsx
onClick={(e) => { e.stopPropagation(); onRemove(ticker); }}
```

This prevents the click event from bubbling up to the parent `<div>` which has `onClick={() => onSelect?.(ticker)}` (line 25). Without this fix, clicking the remove "x" button triggers BOTH `onRemove(ticker)` AND `onSelect(ticker)`, causing a brief unnecessary ticker selection before the removal completes.

The fix is safe because:
- The only parent click handler is the row's `onSelect` — exactly what we want to prevent
- No other functionality depends on event bubbling from this button
- The `onRemove` callback still fires normally
  </action>
